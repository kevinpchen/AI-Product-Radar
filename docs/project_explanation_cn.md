# AI Product Radar 说明文档

## 1. 我想做的是什么

我想做的是一个面向商业分析场景的 AI 产品动态追踪工具。它不是简单的新闻聚合，而是把公开信息源里的 AI 产品变化整理成结构化情报，帮助分析者快速回答三个问题：

- 最近谁发布了什么？
- 这件事属于模型、Agent、视频、语音、搜索、企业工作流还是商业化变化？
- 它对产品形态、竞争格局和商业化路径意味着什么？

我对题目中“AI 动态追踪”的定义是：持续关注国内外 AI 产品和平台的一线公开动态，尤其是官方发布、开发者文档、产品更新和模型能力变化，并将这些零散信息转化为可筛选、可比较、可分析的产品情报流。

之所以这样定义，是因为商业分析岗位关注的不是“知道很多新闻”，而是能把信息变成判断。AI 领域变化很快，如果只看二级媒体，很容易滞后；如果只看模型榜单，又容易忽略真正的产品化和商业化信号。因此这个 MVP 优先追踪官方源和开发者源，再用标签、评分和分析字段把信息组织起来。

## 2. 成品主要能实现哪些功能

成品是一个静态网页 demo，打开 `index.html` 即可查看。主要功能如下：

### 动态情报流

页面展示近期 AI 产品动态，包括公司、地区、发布时间、产品类型、摘要、标签、来源链接和影响分。每条动态都补充了“为什么重要”和“商业判断”，避免停留在新闻罗列。

### 筛选和搜索

支持按地区、类型、关键词和趋势标签筛选。例如可以只看“海外 + Agent 工作流”，也可以搜索“视频”“语音”“企业”等关键词。

### 趋势热词

页面会自动统计标签频次，展示 Agent、Video、Voice、Enterprise、Search、Multimodal 等高频主题。点击标签后可以快速聚焦相关动态。

### 商业影响评分

每条动态都有影响分。评分考虑来源权威性、发布时间、产品形态变化、商业化影响和趋势代表性。它不是绝对评价，而是为了帮助分析者优先关注高价值动态。

### 简报和导出

页面支持复制当前筛选结果的简短简报，也支持导出 CSV，便于继续放入表格、PPT 或分析文档。

### 持续追踪脚本

项目包含 `scripts/update_sources.py`，它读取 `data/sources.json` 中配置的公开信息源，抓取页面并生成候选动态和页面快照。这个脚本体现了 demo 不是一次性手工录入，而是可以继续扩展为每日追踪流程。

## 3. 信息源设计

信息源采用“官方源优先、产品源补充、国内外并重”的原则。

海外源包括 Anthropic、OpenAI、Google、Perplexity、Runway、Cursor、xAI、ElevenLabs、Hugging Face 等。它们能较快反映硅谷和海外 AI 产品的一线动态。

国内源包括阿里通义、字节 Seed、百度等公开发布渠道。国内动态重点关注模型能力、应用落地、AI 硬件、云服务和本地生态。

我没有把普通资讯站作为主源，是因为题目强调“更快捕捉到一线一手动态”。二级媒体适合补充背景，但核心追踪应该从官方博客、产品更新页、开发者文档和发布页面开始。

## 4. 我是如何一步步完成的

### 第一步：拆解题目

题目要求包含 demo、简要分析和说明文档。我先把任务拆成三个产出：

- 一个能展示 AI 动态追踪效果的小工具。
- 一段基于追踪结果的趋势分析。
- 一份说明清楚设计思路、功能和实现过程的文档。

### 第二步：定义 MVP 范围

为了在有限时间内做出可演示成品，我选择静态网页形式，而不是复杂后端。静态网页有三个好处：

- 不需要安装依赖，打开即可展示。
- 便于提交压缩包或附件。
- 可以把重点放在信息结构、追踪逻辑和分析判断上。

### 第三步：设计数据结构

我为每条动态设计了以下字段：

- 日期
- 公司
- 地区
- 市场
- 类型
- 标题
- 摘要
- 为什么重要
- 商业判断
- 标签
- 来源名称
- 来源链接
- 影响分

这个结构不是为了“好看”，而是为了让后续筛选、统计和分析都能运行。

### 第四步：整理公开信息

我基于公开信息整理了种子数据，重点选择近期有代表性的 AI 产品动态，例如 Claude Design、Claude Opus 4.7、Grok 语音 API、ChatGPT 购物和企业应用更新、Google Search Live、Perplexity Agent API、Runway Characters、Seedance 2.0、Qwen3.6-Plus 等。

每条动态都保留了原始来源链接，方便回溯。

### 第五步：搭建前端 demo

前端使用原生 HTML、CSS 和 JavaScript 实现，没有引入外部框架。主要实现：

- 数据读取
- 指标计算
- 卡片渲染
- 搜索筛选
- 标签统计
- 简报复制
- CSV 导出

我刻意没有做成资讯门户首页，而是直接把信息看板作为第一屏，因为题目的重点是“追踪工具”而不是宣传页。

### 第六步：加入持续追踪能力

我写了 `scripts/update_sources.py`，它可以：

- 读取公开源配置
- 抓取网页内容
- 生成页面内容哈希
- 判断页面是否发生变化
- 提取候选动态文本
- 输出 `source-snapshots.json` 和 `latest-candidates.json`

这一步的意义是把 demo 从“一次性展示”推进到“可持续运行”的追踪流程。后续可以把这个脚本接入定时任务，再连接 AI 摘要和自动打标。

### 第七步：输出分析和文档

最后基于数据池写出趋势分析，并补充中文说明文档和中英文 README，保证评估者能快速理解项目目标、运行方法、功能和实现过程。

## 5. 主要利用了哪些 AI 工具

完成过程中，AI 工具主要参与了以下环节：

- 需求拆解：把笔试题拆成 demo、分析、说明文档三个交付物。
- 信息整理：辅助从公开资料中提炼产品动态、摘要和标签。
- 产品设计：帮助确定信息流、筛选器、趋势面板和评分字段。
- 前端实现：辅助生成 HTML、CSS、JavaScript 结构，并检查交互逻辑。
- 文档写作：辅助梳理说明文档、README 和趋势分析。
- 质量检查：辅助检查是否存在缺失字段、文案不一致和运行风险。

我对 AI 工具的使用方式不是让它替代判断，而是让它提高信息整理和实现速度。最终哪些动态被选入、如何归类、商业判断怎么写，仍然需要结合题目目标和商业分析视角来决定。

## 6. 当前局限

这个版本是 MVP，因此还有明显可优化空间：

- 当前数据是种子数据加公开源候选抓取，尚未接入完整自动摘要流程。
- 部分官方页面没有稳定 RSS，需要更细的页面解析规则。
- 影响分是轻量规则和人工判断结合，未来可以加入热度、引用数、社交传播和融资信息。
- 尚未接入 X、Reddit、Hacker News、Product Hunt、Discord 等更前沿社区信号。
- 还没有定时任务、邮件推送或企业微信推送。

## 7. 后续优化方向

- 接入定时任务，每天自动抓取公开源。
- 接入 LLM API 自动生成摘要、标签、去重和影响分建议。
- 增加“公司维度”和“赛道维度”的趋势图。
- 增加竞品对比，例如 OpenAI、Anthropic、Google、Perplexity 在 Agent 入口上的差异。
- 增加风险标签，例如版权、安全、监管、数据隐私和商业化冲突。
- 做成可多人协作的轻量情报后台，支持人工确认和评论。

## 8. 基本判断

我认为 AI 产品动态追踪的关键，不是追求信息量最大，而是追求“及时、可信、可分析”。真正有价值的追踪工具应该把一线来源、结构化整理和商业判断连起来。这个 MVP 以最小成本跑通了这个链路：公开源采集、结构化展示、趋势归纳和分析输出。
